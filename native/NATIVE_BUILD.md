# Embedded Stockfish (Android GDExtension)

This folder builds Stockfish **into the app** as a native library, so the game
can drive it in-process on Android (where you can't spawn a Stockfish
subprocess). Desktop keeps using the Stockfish subprocess unless you also build
the desktop library here.

## TL;DR

On a machine with internet + `git`, `curl`, `unzip`, `python3` (and, for the
optional desktop lib, a host C++ compiler):

```bash
cd native
./build.sh
```

That downloads the NDK + SCons + godot-cpp + Stockfish 11, compiles the arm64
`.so` into `../addons/stockfish/bin/`, and writes
`../addons/stockfish/stockfish.gdextension`. Open the project once, then export
the Android APK/AAB and run on a device — Stockfish is now embedded.

> ⚠️ This was authored without a compiler or Android device on hand, so it's
> built to be correct against the real Stockfish 11 source but **has not been
> compiled or run**. Expect a possible small fix on first build (see Troubleshooting).

## How it works

- **`src/sf_runner.cpp`** embeds Stockfish. Its UCI loop normally reads
  `std::cin` and writes `std::cout`; we swap in custom streambufs so we can feed
  it commands and capture its output. Stockfish runs unmodified on its own thread.
- **`src/stockfish_gd.cpp`** is the godot-cpp class `StockfishGD` exposed to
  GDScript: `start()`, `send(cmd)`, `poll_lines()`, `stop()`.
- **`scripts/chess/stockfish_engine.gd`** auto-detects `StockfishGD` (uses it,
  polling each frame) and otherwise falls back to the desktop subprocess, then to
  the built-in GDScript engine. The rest of the game is unchanged.

## Why Stockfish 11 (not the latest)

Modern Stockfish (16/17+) uses NNUE nets (a 40–76 MB file to embed) and a complex
"universal binary" build that's impractical to compile outside its own Makefile.
**Stockfish 11 is classical** (no net → tiny app, trivial `*.cpp` build) and still
~3300 Elo — far beyond any beginner. Its Skill Level (0–20) and MultiPV work the
same, so the game's difficulty + 3-option mechanic are unaffected.

To use a newer Stockfish instead, set `STOCKFISH_TAG` and adapt the init sequence
in `sf_runner.cpp` (newer SF uses a `UCIEngine` class and the `Stockfish`
namespace) plus add NNUE-net handling.

## Layout

```
native/
  src/                       binding sources (committed)
    sf_runner.{h,cpp}          embedded Stockfish + stream redirection
    stockfish_gd.{h,cpp}       godot-cpp class StockfishGD
    register_types.{h,cpp}     GDExtension entry point
  SConstruct                 build recipe (committed)
  build.sh                   one-command bootstrap + build (committed)
  stockfish.gdextension.template  reference (committed; build.sh generates the real one)
  godot-cpp/ stockfish/ android-ndk-*/   downloaded by build.sh (gitignored)
addons/stockfish/
  bin/*.so                   compiled libraries (gitignored)
  stockfish.gdextension      generated by build.sh (gitignored)
```

## Overrides

```bash
ANDROID_NDK_ROOT=~/Android/Sdk/ndk/23.2.8568313 ./build.sh   # use an existing NDK
STOCKFISH_TAG=sf_11 GODOT_CPP_BRANCH=4.5 ./build.sh
BUILD_DESKTOP=0 ./build.sh                                     # skip the desktop lib
```

## Troubleshooting

- **`scons: command not found`** — install it (`pipx install scons` or
  `pip install --user scons`); the script also vendors it if no pip is present.
- **godot-cpp won't build for Android** — usually an NDK mismatch. NDK r23c is
  known-good with godot-cpp 4.x. Point `ANDROID_NDK_ROOT` at it.
- **Stockfish compile error** — if you bumped `STOCKFISH_TAG`, the init sequence
  / namespace in `sf_runner.cpp` likely needs updating to that version's
  `src/main.cpp`. SF11 is the tested-against target.
- **Godot errors "GDExtension library not found" on desktop** — your
  `addons/stockfish/stockfish.gdextension` lists a platform you didn't build for.
  Build the desktop lib too, or delete that file for desktop dev (the game then
  uses the Stockfish subprocess).
- **Verify the .so loads** — open the project; `ClassDB.class_exists("StockfishGD")`
  should be true. The game logs which transport it picked.
